## Steps (sequential)

1. **Domain models.** Add `FxRate(currencyCode, unitsPerEur, fetchedAt)` and `FxFetchResult` (`Success(rates, asOf)` / `Error(message)`).
2. **DB layer.** Add `FxRateEntity` + DAO with `observeAll(): Flow<List<FxRateEntity>>`, `upsertAll(rates: List<FxRateEntity>)`, `get(code: String): FxRateEntity?`. Wire into `AppDatabase` v7 with `MIGRATION_6_7` that just creates the table.
3. **Network.** `EcbFxRatesApi` exposes one method returning `ResponseBody`. `EcbFxRatesSource` parses XML with `Xml.newPullParser()` from `android.util` and returns `FxFetchResult`.
4. **Repository.** `FxRatesRepository.observeRates(): Flow<Map<String, FxRate>>` (maps to domain), `refresh(): FxFetchResult` (calls source, persists rows on success).
5. **Use cases.** `ConvertCurrencyUseCase` is a stateless math helper: takes the current rate map + (amount, from, to). `RefreshFxRatesUseCase` calls `repository.refresh()`.
6. **Settings.** Add `displayCurrencyCode` (SharedPreferences key `display_currency_code`). Picker UI in Settings — reuses the existing currency-picker bottom sheet if available; otherwise a small `AlertDialog` listing common codes (EUR, USD, GBP, BRL, CHF). Decision: a `DropdownMenu` next to the row, populated from the cached `fx_rates` table + EUR (so the user can only pick what's actually convertible).
7. **TotalWealth + use case.** Extend the domain model and update `GetTotalWealthUseCase` to combine the three flows and compute `totalInDisplayCurrency` lazily. If any per-currency total can't be converted, the converted total is `null`.
8. **Header UI.** When `totalInDisplayCurrency != null && isMixedCurrency`, show the converted total as the headline and demote the per-currency lines to a sub-list. Footnote: "FX as of <YYYY-MM-DD>" derived from `fxFetchedAt`. When `totalInDisplayCurrency == null && isMixedCurrency`, show the existing per-currency breakdown plus "FX rates unavailable — refresh prices to update".
9. **Worker.** `RefreshPricesWorker.doWork()` runs the FX refresh after Euribor with `runCatching`. Logs only.
10. **Tests.**
    - `EcbFxRatesSourceTest` with MockWebServer serving a canned ECB XML body — assert parsed rates + asOf.
    - `ConvertCurrencyUseCaseTest`: identity, EUR↔X, X↔Y, missing rate.
    - `FxRatesRepositoryTest`: refresh persists; observeRates emits.
    - VM unit test (one test) confirming `TotalWealth.totalInDisplayCurrency` flows into the UI state.

## Risks

- **XML parsing edge cases.** ECB namespaces every element with `gesmes:`/`xmlns=...`. Use `name == "Cube"` (no namespace fight) and let the parser ignore unknown.
- **Hilt module wiring.** `EcbFxRatesApi` needs its own `Retrofit` (different base URL). Easy with `@Named("ecb")` qualifiers or a separate `@Provides Retrofit` named function.
- **Schema export.** First Room rebuild generates `schemas/.../7.json`. Verify it appears and commit it.
- **Avoid touching `currencyCode`.** Users may have `currencyCode = "EUR"` and we don't want the picker to overwrite it. The picker only changes `displayCurrencyCode`.
